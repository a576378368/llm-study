# 第三章：微调技术

> 微调是将预训练模型适配到特定任务的关键步骤。本章详细介绍参数高效微调、监督微调、对比微调等技术。

## 3.1 参数高效微调

参数高效微调（Parameter-Efficient Fine-Tuning，PEFT）旨在用少量参数实现模型适配，避免全量微调的巨大计算开销。

### 3.1.1 LoRA技术详解

LoRA（Low-Rank Adaptation）是最流行的参数高效微调方法之一。

**核心思想**：

LoRA假设预训练模型的权重更新具有低秩特性。因此，不直接更新完整权重矩阵，而是学习两个低秩矩阵的乘积来表示权重更新。

**数学原理**：

对于预训练权重矩阵 $W_0 \in \mathbb{R}^{d \times k}$，LoRA学习一个低秩分解：

$$W = W_0 + \Delta W = W_0 + BA$$

其中：
- $B \in \mathbb{R}^{d \times r}$
- $A \in \mathbb{R}^{r \times k}$
- $r \ll \min(d, k)$ 为秩

前向传播时：
$$h = W_0 x = W_0 x + BAx$$

**代码实现**：
```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class LoRALinear(nn.Module):
    """LoRA线性层"""
    def __init__(self, in_features, out_features, rank=4, alpha=1.0, dropout=0.0):
        super().__init__()
        self.rank = rank
        self.alpha = alpha
        self.scaling = alpha / rank

        # 冻结原始权重
        self.weight = nn.Parameter(torch.zeros(out_features, in_features), requires_grad=False)

        # LoRA参数
        self.lora_A = nn.Parameter(torch.randn(rank, in_features) / np.sqrt(rank))
        self.lora_B = nn.Parameter(torch.zeros(out_features, rank))

        if dropout > 0:
            self.lora_dropout = nn.Dropout(dropout)
        else:
            self.lora_dropout = nn.Identity()

    def forward(self, x):
        # 原始权重输出
        base_output = F.linear(x, self.weight)

        # LoRA输出
        lora_output = F.linear(self.lora_dropout(x), self.lora_B @ self.lora_A) * self.scaling

        return base_output + lora_output

    def merge_weights(self):
        """合并LoRA权重到原始权重"""
        W = self.weight.data + (self.lora_B @ self.lora_A) * self.scaling
        self.weight.data = W
        self.rank = 0  # 标记为已合并

class LoRAEmbedding(nn.Module):
    """LoRA嵌入层"""
    def __init__(self, num_embeddings, embedding_dim, rank=4, alpha=1.0):
        super().__init__()
        self.rank = rank
        self.alpha = alpha

        # 冻结原始嵌入
        self.weight = nn.Parameter(torch.zeros(num_embeddings, embedding_dim), requires_grad=False)

        # LoRA参数
        self.lora_A = nn.Parameter(torch.randn(rank, num_embeddings) / np.sqrt(rank))
        self.lora_B = nn.Parameter(torch.randn(num_embeddings, rank))

    def forward(self, x):
        base_output = F.embedding(x, self.weight)
        lora_output = F.embedding(x, self.lora_B @ self.lora_A) * (self.alpha / self.rank)
        return base_output + lora_output

class LoRAConfig:
    """LoRA配置"""
    def __init__(
        self,
        rank=4,
        lora_alpha=16,
        target_modules=None,
        lora_dropout=0.05,
        bias='none'
    ):
        self.rank = rank
        self.lora_alpha = lora_alpha
        self.target_modules = target_modules or ['q_proj', 'v_proj']
        self.lora_dropout = lora_dropout
        self.bias = bias

class LoRAModel(nn.Module):
    """带LoRA的模型"""
    def __init__(self, base_model, config):
        super().__init__()
        self.base_model = base_model
        self.config = config
        self.lora_layers = []

        self._apply_lora()

    def _apply_lora(self):
        """在指定模块上应用LoRA"""
        for name, module in self.base_model.named_modules():
            for target_module in self.config.target_modules:
                if target_module in name:
                    self._replace_with_lora(module, name)

    def _replace_with_lora(self, module, name):
        """替换模块为LoRA版本"""
        if isinstance(module, nn.Linear):
            lora_linear = LoRALinear(
                module.in_features,
                module.out_features,
                rank=self.config.rank,
                alpha=self.config.lora_alpha,
                dropout=self.config.lora_dropout
            )
            # 复制原始权重
            lora_linear.weight.data = module.weight.data.clone()
            setattr(self.base_model, name, lora_linear)
            self.lora_layers.append(lora_linear)

        elif isinstance(module, nn.Embedding):
            lora_embedding = LoRAEmbedding(
                module.num_embeddings,
                module.embedding_dim,
                rank=self.config.rank,
                alpha=self.config.lora_alpha
            )
            lora_embedding.weight.data = module.weight.data.clone()
            setattr(self.base_model, name, lora_embedding)
            self.lora_layers.append(lora_embedding)

    def train(self, mode=True):
        self.base_model.train(mode)
        # LoRA层始终可训练
        for layer in self.lora_layers:
            layer.requires_grad_(True)

    def get_trainable_params(self):
        """获取可训练参数"""
        trainable = []
        for layer in self.lora_layers:
            trainable.extend(list(layer.parameters()))
        return trainable

    def merge_and_unload(self):
        """合并LoRA权重并卸载LoRA"""
        for layer in self.lora_layers:
            layer.merge_weights()
        self.lora_layers = []
        return self.base_model
```

**训练配置**：
```python
def train_with_lora(model, config, train_loader, optimizer, num_epochs):
    """LoRA训练循环"""
    trainable_params = model.get_trainable_params()

    # 只优化LoRA参数
    optimizer = torch.optim.AdamW(trainable_params, lr=1e-3, weight_decay=0.01)

    for epoch in range(num_epochs):
        for batch in train_loader:
            optimizer.zero_grad()

            outputs = model(**batch)
            loss = outputs.loss

            loss.backward()
            torch.nn.utils.clip_grad_norm_(trainable_params, 1.0)
            optimizer.step()

            print(f"Epoch {epoch}, Loss: {loss.item():.4f}")

    # 合并权重
    model.merge_and_unload()
    return model
```

### 3.1.2 Prefix Tuning

Prefix Tuning在输入前添加可学习的前缀来调整模型行为。

**核心思想**：

不在模型权重上做微调，而是在每层的输入序列前添加连续的可学习向量（prefix）。

**代码实现**：
```python
class PrefixTuning(nn.Module):
    """Prefix Tuning实现"""
    def __init__(self, num_layers, num_heads, prefix_len=20, d_model=768):
        super().__init__()
        self.prefix_len = prefix_len

        # 可学习的prefix参数
        self.prefix_tokens = nn.Parameter(
            torch.randn(1, prefix_len, d_model) * 0.02
        )

        # 重投影层（用于将prefix适配到各层）
        self.prefix_proj = nn.Sequential(
            nn.Linear(d_model, d_model * 2),
            nn.ReLU(),
            nn.Linear(d_model * 2, d_model * 2)  # 用于QKV
        )

    def forward(self, hidden_states):
        batch_size = hidden_states.size(0)

        # 扩展prefix到batch大小
        prefix = self.prefix_tokens.expand(batch_size, -1, -1)

        # 重投影
        prefix = self.prefix_proj(prefix)

        # 拼接prefix和输入
        hidden_states = torch.cat([prefix, hidden_states], dim=1)

        return hidden_states

class PrefixTuningConfig:
    """Prefix Tuning配置"""
    def __init__(
        self,
        prefix_len=20,
        num_virtual_tokens=20,
        projection_size=512,
        hidden_size=768,
        num_layers=12
    ):
        self.prefix_len = prefix_len
        self.num_virtual_tokens = num_virtual_tokens
        self.projection_size = projection_size
        self.hidden_size = hidden_size
        self.num_layers = num_layers
```

### 3.1.3 P-Tuning

P-Tuning（Prompt Tuning的改进版）使用可学习的连续提示。

**与Prefix Tuning的区别**：

- P-Tuning专注于输入层
- 使用LSTM对prompt进行编码以增加表达能力
- 添加了anchor tokens提升学习效果

**代码实现**：
```python
class P-tuning(nn.Module):
    """P-Tuning实现"""
    def __init__(self, d_model, num_prompt_tokens=16, hidden_size=128):
        super().__init__()
        self.num_prompt_tokens = num_prompt_tokens

        # 可学习的prompt embedding
        self.prompt_embeddings = nn.Parameter(
            torch.randn(1, num_prompt_tokens, d_model) * 0.02
        )

        # LSTM编码器
        self.lstm = nn.LSTM(
            input_size=d_model,
            hidden_size=hidden_size,
            num_layers=2,
            bidirectional=True,
            batch_first=True,
            dropout=0.1
        )

        # 投影回原始维度
        self.output_proj = nn.Linear(hidden_size * 2, d_model)

    def forward(self, input_embeddings):
        batch_size = input_embeddings.size(0)

        # 获取prompt
        prompt = self.prompt_embeddings.expand(batch_size, -1, -1)

        # LSTM编码
        prompt_encoded = self.lstm(prompt)[0]
        prompt_encoded = self.output_proj(prompt_encoded)

        # 拼接
        output = torch.cat([prompt_encoded, input_embeddings], dim=1)

        return output

class PromptEncoder(nn.Module):
    """Prompt编码器"""
    def __init__(self, num_tokens, embedding_dim, hidden_size):
        super().__init__()
        self.embedding = nn.Embedding(num_tokens, embedding_dim)

        self.lstm = nn.LSTM(
            embedding_dim, hidden_size,
            num_layers=2, bidirectional=True, batch_first=True
        )

        self.linear = nn.Linear(hidden_size * 2, embedding_dim)

    def forward(self, prompt_tokens):
        embedded = self.embedding(prompt_tokens)
        encoded = self.lstm(embedded)[0]
        return self.linear(encoded)
```

### 3.1.4 Prompt Tuning

Prompt Tuning是最简化的prefix tuning，只在输入层添加可学习的提示。

**核心思想**：

仅添加少量可学习参数作为prompt，无需改变模型结构。

```python
class PromptTuning(nn.Module):
    """简化的Prompt Tuning"""
    def __init__(self, num_prompts=5, embedding_dim=768):
        super().__init__()
        self.prompt_embeddings = nn.Parameter(
            torch.randn(num_prompts, embedding_dim) * 0.02
        )

    def forward(self, input_embeddings):
        # 拼接prompt到输入
        prompt = self.prompt_embeddings.unsqueeze(0).expand(input_embeddings.size(0), -1, -1)
        return torch.cat([prompt, input_embeddings], dim=1)

# 使用示例
prompt_tuning = PromptTuning(num_prompts=20, embedding_dim=768)

# 在推理时
input_ids = tokenizer("这是一个测试", return_tensors='pt')
input_embeddings = model.get_input_embeddings()(input_ids['input_ids'])
input_embeddings = prompt_tuning(input_embeddings)
```

### 3.3.5 Adapter等方法

Adapter在Transformer层之间插入小型瓶颈网络。

**Adapter结构**：
```
Input → Linear(d_model → d_adapter) → ReLU → Linear(d_adapter → d_model) → Output
                                      ↓
                              残差连接
```

**代码实现**：
```python
class Adapter(nn.Module):
    """Adapter模块"""
    def __init__(self, d_model, d_adapter=64, activation='relu'):
        super().__init__()
        self.d_model = d_model
        self.d_adapter = d_adapter

        self.down_project = nn.Linear(d_model, d_adapter)
        self.up_project = nn.Linear(d_adapter, d_model)

        if activation == 'relu':
            self.activation = nn.ReLU()
        elif activation == 'gelu':
            self.activation = nn.GELU()

        # 初始化
        nn.init.normal_(self.down_project.weight, std=1e-3)
        nn.init.zeros_(self.up_project.weight)
        nn.init.zeros_(self.down_project.bias)
        nn.init.zeros_(self.up_project.bias)

    def forward(self, x):
        # Adapter前向
        adapter_output = self.up_project(self.activation(self.down_project(x)))

        # 残差连接
        return x + adapter_output

class AdapterConfig:
    """Adapter配置"""
    def __init__(
        self,
        d_adapter=64,
        adapter_activation='relu',
        adapter_layer_norm=True,
        skip_layer_norm=False
    ):
        self.d_adapter = d_adapter
        self.adapter_activation = adapter_activation
        self.adapter_layer_norm = adapter_layer_norm
        self.skip_layer_norm = skip_layer_norm

class TransformerWithAdapters(nn.Module):
    """带Adapter的Transformer"""
    def __init__(self, base_transformer, config):
        super().__init__()
        self.base_transformer = base_transformer
        self.adapters = nn.ModuleList([
            Adapter(base_transformer.d_model, config.d_adapter, config.adapter_activation)
            for _ in range(base_transformer.num_layers)
        ])

    def forward(self, x, **kwargs):
        for i, layer in enumerate(self.base_transformer.layers):
            x = layer(x, **kwargs)
            # 在每层后添加Adapter
            x = self.adapters[i](x)
        return x
```

**流程概述**：
```

```

```python
            'input_ids': input_ids,
            'attention_mask': attention_mask,
            'labels': labels
        }
```

```python
    """文本分类微调"""

    model.classifier = nn.Linear(model.config.hidden_size, num_labels)

    optimizer = torch.optim.AdamW(model.parameters(), lr=learning_rate)

    for epoch in range(3):
        model.train()
        total_loss = 0

        for batch in train_dataset:
            optimizer.zero_grad()

            outputs = model(
                input_ids=batch['input_ids'],
                attention_mask=batch['attention_mask']
            )

            loss = nn.CrossEntropyLoss()(
                outputs.logits,
                batch['labels']
            )

            loss.backward()
            optimizer.step()

            total_loss += loss.item()

        print(f"Epoch {epoch}, Loss: {total_loss / len(train_dataset):.4f}")

    return model
```

```python
class QADataset(torch.utils.data.Dataset):
    """问答数据集"""
    def __init__(self, contexts, questions, answers, tokenizer, max_length=512):
        self.contexts = contexts
        self.questions = questions
        self.answers = answers
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __len__(self):
        return len(self.contexts)

    def __getitem__(self, idx):
        context = self.contexts[idx]
        question = self.questions[idx]
        answer = self.answers[idx]

        encoding = self.tokenizer(
            question,
            context,
            max_length=self.max_length,
            padding='max_length',
            truncation='only_second',
            return_tensors='pt',
            return_offsets_mapping=True
        )

        offset_mapping = encoding.pop('offset_mapping').squeeze()
        start_pos, end_pos = self._find_answer_position(
            answer, context, offset_mapping
        )

        return {
            'input_ids': encoding['input_ids'].squeeze(),
            'attention_mask': encoding['attention_mask'].squeeze(),
            'start_positions': torch.tensor(start_pos),
            'end_positions': torch.tensor(end_pos)
        }

    def _find_answer_position(self, answer, context, offset_mapping):
        """找到答案在token序列中的位置"""
        start_char = context.find(answer)
        if start_char == -1:
            return 0, 0

        end_char = start_char + len(answer)

        start_token = 0
        end_token = 0

        for idx, (start, end) in enumerate(offset_mapping):
            if start <= start_char < end:
                start_token = idx
            if start < end_char <= end:
                end_token = idx
                break

        return start_token, end_token
```

## 3.3 对比微调

对比微调通过对比学习损失优化表示，适用于检索、匹配等任务。

### 3.3.1 对比学习基础

**对比学习核心思想**：

通过拉近相似样本、推远不相似样本来学习表示。

$$\mathcal{L} = -\log \frac{\exp(sim(z_i, z_j^+) / \tau)}{\sum_{k=1}^{N} \exp(sim(z_i, z_k) / \tau)}$$

**SimCLR框架**：
```
输入x → 编码器f(·) → 表示h → 投影头g(·) → 表示z
                                          ↓
                            正样本对(x_i, x_j)
                            负样本对(x_i, x_k)
```

### 3.3.2 对比预训练与微调

**对比预训练**：
```python
class ContrastivePretraining:
    """对比预训练"""
    def __init__(self, model, temperature=0.05):
        self.model = model
        self.temperature = temperature

    def info_nce_loss(self, z_i, z_j):
        """
        计算InfoNCE损失
        z_i, z_j: 来自同一样本的两个视图
        """
        batch_size = z_i.size(0)

        # 归一化
        z_i = nn.functional.normalize(z_i, dim=1)
        z_j = nn.functional.normalize(z_j, dim=1)

        # 相似度矩阵
        similarity = torch.matmul(z_i, z_j.T) / self.temperature

        # 标签：正样本在对角线
        labels = torch.arange(batch_size, device=z_i.device)

        loss = nn.CrossEntropyLoss()(similarity, labels)
        return loss

    def forward(self, batch):
        # 两个增强视图
        x_i, x_j = batch['view1'], batch['view2']

        h_i = self.model(x_i)  # 编码器输出
        h_j = self.model(x_j)

        z_i = self.projection_head(h_i)  # 投影
        z_j = self.projection_head(h_j)

        loss = self.info_nce_loss(z_i, z_j)
        return loss
```

**Sentence-BERT对比微调**：
```python
class SBERTfine-tuning:
    """Sentence-BERT微调"""
    def __init__(self, model, tokenizer):
        self.model = model
        self.tokenizer = tokenizer

    def soft_triplet_loss(self, embedding, label, margin=0.5):
        """
        软三元组损失
        embedding: (batch_size, hidden_dim)
        label: (batch_size,)
        """
        # 计算相似度矩阵
        similarity = torch.matmul(embedding, embedding.T)

        # 找到正样本和负样本
        mask = torch.eq(label.unsqueeze(1), label.unsqueeze(0))  # 同类为True

        # 正样本损失
        positive_loss = (margin - similarity[mask]).clamp(0).pow(2).mean()

        # 负样本损失
        negative_loss = (similarity[~mask] - margin).clamp(0).pow(2).mean()

        return positive_loss + negative_loss

    def contrastive_loss(self, embedding, label, temperature=0.05):
        """对比损失"""
        # 归一化
        embedding = nn.functional.normalize(embedding, dim=1)

        # 相似度
        similarity = torch.matmul(embedding, embedding.T) / temperature

        # 正样本：同类别
        labels = label.contiguous().view(-1, 1)
        mask = torch.eq(labels, labels.T).float()

        # InfoNCE
        exp_sim = torch.exp(similarity)
        log_prob = similarity - torch.log(exp_sim.sum(1, keepdim=True))

        # 只取正样本
        mask_pos = mask
        loss = -(mask_pos * log_prob).sum() / mask_pos.sum()

        return loss

    def train_step(self, sentences, labels):
        # 编码
        outputs = self.model(**self.tokenizer(sentences, return_tensors='pt', padding=True))
        embedding = outputs.last_hidden_state[:, 0]  # [CLS]

        # 计算损失
        loss = self.contrastive_loss(embedding, labels)

        return loss
```

## 3.4 微调策略

### 3.4.1 全量微调 vs 高效微调

**全量微调**：

| 优点 | 缺点 |
|------|------|
| 模型能力完全释放 | 计算资源大 |
| 任务适配性好 | 容易过拟合 |
| 无需额外参数 | 存储成本高 |

**高效微调**：

| 优点 | 缺点 |
|------|------|
| 参数少（万级） | 可能损失部分能力 |
| 训练快 | 调参敏感 |
| 存储成本低 | 不适合所有任务 |

**选择建议**：

- 算力充足、任务复杂：全量微调
- 算力有限、任务简单：LoRA/QLoRA
- 多任务场景：Prefix Tuning/Prompt Tuning

### 3.4.2 微调时机选择

**何时微调**：

1. **新任务适配**：预训练模型不具备该能力
2. **领域适应**：需要适配特定行业知识
3. **指令对齐**：如InstructGPT的RLHF

**训练数据量与策略选择**：
```
数据量
  │
  │← 极少(百级) → Prompt Tuning / Few-shot
  │
  │← 少量(千级) → LoRA / Adapter
  │
  │← 中等(万级) → 全量微调
  │
  └────────────────────────→ 更多数据
```

### 3.4.3 微调数据构建

**高质量数据要点**：

1. **质量优先于数量**：错误数据会损害模型
2. **多样性**：覆盖不同场景和表达方式
3. **格式一致**：保持训练数据格式统一
4. **长度适中**：避免过长或过短的样本

**数据清洗**：
```python
class DataFilter:
    """微调数据清洗"""
    def __init__(self, min_length=10, max_length=2048):
        self.min_length = min_length
        self.max_length = max_length

    def filter_by_length(self, texts):
        """长度过滤"""
        return [t for t in texts if self.min_length <= len(t) <= self.max_length]

    def filter_by_quality(self, texts):
        """质量过滤"""
        valid = []
        for text in texts:
            # 移除重复句子过多的文本
            sentences = text.split('。')
            if len(sentences) != len(set(sentences)):
                continue
            # 移除特殊字符过多的文本
            if sum(1 for c in text if c in '!?@#$%') / len(text) > 0.1:
                continue
            valid.append(text)
        return valid

    def deduplicate(self, texts):
        """去重"""
        seen = set()
        unique = []
        for t in texts:
            if t not in seen:
                seen.add(t)
                unique.append(t)
        return unique
```

## 本章小结

本章介绍了微调的核心技术：

1. **参数高效微调**：
   - **LoRA**：学习低秩矩阵更新，效率高，效果好
   - **Prefix/Prompt Tuning**：在输入或中间添加可学习提示
   - **Adapter**：瓶颈结构，对原模型影响小

2. **监督微调**：
   - 数据格式和模板设计
   - 不同任务的适配方法
   - prompt工程在微调中的应用

3. **对比微调**：
   - 基于对比学习优化表示
   - 适用于检索、匹配等任务
   - InfoNCE和三元组损失

4. **微调策略选择**：
   - 根据算力和任务选择合适方法
   - 数据质量和多样性至关重要
   - 合理的数据清洗和构造
