第三章：大模型基础理论#

本章介绍大模型的核心概念、预训练理论和微调理论，是理解现代大语言模型的基础。

3.1 大模型概念#

3.1.1 什么是大模型#

大模型是指参数规模巨大、在海量数据上训练的深度学习模型。随着计算能力和数据量的增长，模型规模不断增大。

模型规模的演进：

早期：几万到几十万参数
2012-2015：数百万参数（AlexNet）
2017-2018：十亿参数级（BERT, GPT-2）
2019-2020：百亿参数级（GPT-3, BERT-Large）
2021-2022：千亿参数级（GPT-3, PaLM）
2023+：万亿参数级（GPT-4, Claude, Llama 3）

参数规模：

  • 小模型：< 1亿参数

  • 中模型：1亿 - 10亿参数

  • 大模型：10亿 - 100亿参数

  • 超大模型：> 100亿参数

3.1.2 为什么大模型更好#

大模型展现出许多优势：

1. 零样本和少样本能力

  • 不需要大量标注数据就能完成任务

  • 在新任务上表现良好

2. 强泛化能力

  • 在一个任务上学到的知识可以迁移到其他任务

  • 适应各种下游应用

3. 上下文理解

  • 能理解更长的上下文

  • 更好地理解复杂关系

4. 推理能力

  • 更强的逻辑推理能力

  • 能够进行多步推理

5. 多样性

  • 生成的输出更多样化

  • 更少重复和死板

6. 工具使用

  • 更好地使用外部工具

  • 更强的环境交互能力

3.1.3 算力需求与成本#

大模型的训练和部署需要大量算力。

训练成本：

  • GPU/TPU数量：几十到几千张

  • 训练时间：几天到几个月

  • 电费：数万到数十万美元

  • 人力成本：研究团队、数据标注等

部署成本：

  • 推理硬件：高性能GPU

  • 云服务费用：按使用量计费

  • 维护成本：模型更新、监控等

成本优化：

  • 模型量化

  • 模型蒸馏

  • 混合精度训练

  • 分布式推理

3.2 预训练理论#

3.2.1 自监督学习#

自监督学习是一种无需人工标注的训练方法。

原理：

从数据本身构造监督信号
数据 → 自监督任务 → 预训练模型

特点：

  • 数据丰富：互联网文本、代码等

  • 无需标注：节省人力成本

  • 大规模：可以训练数十亿参数的模型

应用：

  • 文本：掩码语言模型、预测任务

  • 代码：代码补全、缺陷检测

  • 图像：图像分类、目标检测

3.2.2 预训练目标设计#

预训练目标决定了模型学到了什么。

常见预训练目标：

1. 掩码语言模型 (MLM)

随机掩码15%的词
预测被掩码的词

优势：

  • 双向上下文

  • 不需要移动窗口

  • 更好的表示学习

2. 下一句预测 (NSP)

判断两句是否连续

3. 文本生成

给定上文，预测下一个词

4. 填空任务

预测缺失的句子

5. 对比学习

拉近相同类样本，推远不同类样本

3.2.3 数据集构建#

高质量数据集是预训练成功的关键。

数据来源：

  • 公开网页：Wikipedia、Common Crawl等

  • 代码：GitHub、代码仓库

  • 文档：学术论文、技术文档

  • 对话数据：聊天记录、客服记录

数据预处理：

  1. 数据收集

    • 抓取海量文本

    • 去重和过滤

  2. 数据清洗

    • 移除噪音

    • 过滤低质量内容

    • 处理特殊字符

  3. 分词

    • 构建词汇表

    • 子词分词（BPE、WordPiece）

  4. 数据过滤

    • 质量过滤

    • 安全过滤

    • 语言检测

数据集规模：

  • 小数据集：数GB到数十GB

  • 中等数据集：数十GB到数百GB

  • 大数据集：数百GB到数TB

  • 超大规模：数TB到数十TB

数据分布：

  • 互联网文本

  • 专业领域内容

  • 多语言内容

3.2.4 训练策略#

预训练过程需要精心设计。

训练流程：

1. 数据准备
2. 模型初始化
3. 分布式训练
4. 梯度累积
5. 混合精度训练
6. 模型保存
7. 验证和评估

分布式训练：

  • 数据并行：复制模型，分批训练

  • 模型并行：拆分模型到多设备

  • 混合并行：结合两种方法

梯度累积：

累积多个batch的梯度后更新
模拟大batch size训练

混合精度训练：

使用FP16和FP32混合
加速训练，减少显存

学习率策略：

预热：前几个step从低学习率开始
主训练：正常学习率训练
衰减：学习率逐渐衰减

3.2.5 预训练优化#

为了提高预训练效果，需要考虑各种优化。

模型初始化：

  • 合理的初始化范围

  • Xavier初始化

  • He初始化

训练稳定性：

  • 梯度裁剪

  • 批归一化

  • 梯度检查点

泛化能力：

  • 正则化方法

  • 数据增强

  • 知识蒸馏

效率优化：

  • Flash Attention

  • 优化算子

  • 减少显存占用

3.3 微调理论#

3.3.1 微调vs从头训练#

微调是在预训练模型基础上进行微调，vs从头训练。

微调的优势：

  • 节省时间：不需要从头训练

  • 节省算力：不需要大量资源

  • 更好效果：基于已有知识

  • 易于实现：使用现成模型

从头训练：

  • 需要大量数据

  • 需要大量算力

  • 训练时间长

  • 容易过拟合

适用场景：

  • 微调：大多数场景，特别是有少量数据时

  • 从头训练：特殊领域、小数据集、特定需求

3.3.2 微调方法分类#

根据参数更新范围，分为：

1. 全量微调

更新所有参数
适用于参数量小的模型

问题：

  • 需要大量显存

  • 可能过拟合

  • 存储成本高

2. 参数高效微调 (PEFT)

只更新少量参数
适用于大模型

常见方法：

  • LoRA

  • Prefix Tuning

  • P-Tuning

  • Adapter

  • Prompt Tuning

3.3.3 微调中的问题与解决方案#

问题1：灾难性遗忘

模型在学习新任务时忘记旧任务

解决方案：

  • 持续学习

  • 冻结部分参数

  • 回忆机制

问题2：领域适配

预训练领域与目标任务领域差异大

解决方案：

  • 领域自适应预训练

  • 领域特定微调

  • 多任务学习

问题3：计算资源限制

大模型难以进行全量微调

解决方案：

  • PEFT方法

  • 量化

  • 剪枝

问题4：任务适配性

通用模型不擅长特定任务

解决方案：

  • 设计特定提示

  • 任务特定的微调数据

  • 任务特定的层

3.3.4 参数高效微调详解#

1. LoRA (Low-Rank Adaptation)

原理：

冻结预训练模型参数
添加低秩分解矩阵
只有少量参数可训练

W = W₀ + ΔW = W₀ + BA

B: (d, r)
A: (r, d)

r << d，r为秩

优势：

  • 极少可训练参数（<1%）

  • 保持模型性能

  • 易于合并和部署

实现：

class LoRALinear(nn.Module):
    def __init__(self, original_linear, rank, alpha):
        super().__init__()
        self.original_linear = original_linear

        # 冻结原始参数
        for param in original_linear.parameters():
            param.requires_grad = False

        # LoRA参数
        self.rank = rank
        self.alpha = alpha

        self.lora_A = nn.Parameter(torch.randn(rank, original_linear.in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.randn(original_linear.out_features, rank) * 0.01)

    def forward(self, x):
        # 原始前向
        output = self.original_linear(x)

        # LoRA部分
        lora_output = self.alpha * (x @ self.lora_A.T @ self.lora_B.T)

        return output + lora_output

2. Prefix Tuning

原理：

在输入和输出前添加可训练的虚拟token
这些token作为前缀影响模型行为

3. Prompt Tuning

原理：

将提示词参数化
只训练提示词，不训练模型

4. Adapter

原理：

在Transformer层中插入小型适配模块
只有适配模块参数可训练

3.3.5 微调策略#

1. 监督微调

流程：

构建任务数据集

有监督微调
    ↓
模型评估

2. 对比微调

利用对比学习：

拉近相似样本
推远不同样本

3. RLHF微调

结合强化学习：

人类反馈强化学习
奖励建模
策略优化

3.3.6 微调时机选择#

1. 数据量少

在数据充足时微调

2. 领域差异大

先进行领域自适应
再进行任务微调

3. 特定任务

针对任务需求设计微调策略

# LoRA实现示例
import torch
import torch.nn as nn

class LinearWithLoRA(nn.Module):
    def __init__(self, in_features, out_features, rank=16, alpha=1.0):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features)
        self.rank = rank
        self.alpha = alpha

        # 冻结原始线性层
        for param in self.linear.parameters():
            param.requires_grad = False

        # LoRA参数
        self.lora_A = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_features, rank))

        # 使用缩放因子
        self.scaling = self.alpha / self.rank

    def forward(self, x):
        # 原始线性计算
        output = self.linear(x)

        # LoRA计算
        lora_output = torch.matmul(x, self.lora_A.T) @ self.lora_B.T
        lora_output = lora_output * self.scaling

        return output + lora_output

    def __init__(self, model):
        super().__init__()
        self.model = model
        self.task_head = nn.Linear(model.config.hidden_size, num_labels)

        # 冻结模型大部分参数
        for param in self.model.parameters():
            param.requires_grad = False

        # 只解冻部分层
        for name, param in self.model.named_parameters():
            if 'encoder.layers.11' in name or 'encoder.layers.12' in name:
                param.requires_grad = True

    def forward(self, input_ids, attention_mask):
        outputs = self.model(input_ids=input_ids, attention_mask=attention_mask)
        hidden_states = outputs.last_hidden_state[:, 0]  # [CLS] token
        logits = self.task_head(hidden_states)

        return logits

# 预训练目标示例：MLM
class MLMModel(nn.Module):
    def __init__(self, vocab_size, d_model, num_layers=6):
        super().__init__()
        self.token_embedding = nn.Embedding(vocab_size, d_model)
        self.position_embedding = nn.Embedding(512, d_model)  # 上下文长度
        self.layers = nn.ModuleList([
            nn.TransformerEncoderLayer(d_model=d_model, nhead=8, batch_first=True)
            for _ in range(num_layers)
        ])
        self.output = nn.Linear(d_model, vocab_size)

    def forward(self, input_ids):
        batch_size, seq_len = input_ids.shape
        positions = torch.arange(seq_len, device=input_ids.device).unsqueeze(0).expand(batch_size, -1)

        x = self.token_embedding(input_ids) + self.position_embedding(positions)
        x = x.transpose(1, 2)  # (batch, seq_len) -> (batch, d_model, seq_len)

        for layer in self.layers:
            x = layer(x).transpose(1, 2)  # 回到 (batch, seq_len, d_model)

        logits = self.output(x)
        return logits

# 使用示例
vocab_size = 30522
d_model = 768

model = MLMModel(vocab_size, d_model)
mlm_input = torch.randint(0, vocab_size, (32, 128))  # batch_size=32, seq_len=128
outputs = model(mlm_input)

print(f"Input shape: {mlm_input.shape}")      # (32, 128)
print(f"Output shape: {outputs.shape}")      # (32, 128, 30522)
