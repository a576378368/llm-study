第二章：深度学习进阶#

本章深入介绍现代深度学习架构、稠密表示学习和序列建模技术，为理解Transformer和大模型打下基础。

2.1 现代深度学习架构#

2.1.1 卷积神经网络 (CNN)#

卷积神经网络是计算机视觉领域的基石，其卷积操作能够有效捕捉图像的局部特征。

卷积操作：

输出 = 输入 ⊛ 滤波器 + 偏置

其中 ⊛ 表示卷积运算

核心组件：

  1. 卷积层

    • 特征提取：通过卷积核提取局部特征

    • 权重共享：减少参数数量

    • 步长：控制输出大小

    • 填充：保持空间维度

  2. 池化层

    • 下采样：减少计算量和参数

    • 常用类型：

      • 最大池化：保留最强特征

      • 平均池化：平滑特征

  3. 经典架构：

    • LeNet-5：早期卷积网络

    • AlexNet：深度学习复兴

    • VGG：小卷积核堆叠

    • GoogLeNet：Inception模块

    • ResNet：残差连接

CNN在NLP中的应用：

  • 文本卷积：一维卷积处理序列

  • 文档分类

  • 情感分析

2.1.2 循环神经网络 (RNN)#

循环神经网络专门设计用于处理序列数据，能够记住历史信息。

RNN结构：

h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b_h)

h_t：当前时刻的隐藏状态
h_{t-1}：上一时刻的隐藏状态
x_t：当前时刻的输入
W_hh, W_xh：权重矩阵
b_h：偏置

问题：

  • 梯度消失：长序列信息难以传递

  • 梯度爆炸：可能导致训练不稳定

  • 并行性差：序列依赖顺序计算

改进方法：

  1. LSTM (长短期记忆网络)

    遗忘门：f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
输入门：i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
输出门：o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
候选细胞状态：C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)
细胞状态：C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
隐藏状态：h_t = o_t ⊙ tanh(C_t)

  2. GRU (门控循环单元)

    重置门：r_t = σ(W_r · [h_{t-1}, x_t])
更新门：z_t = σ(W_z · [h_{t-1}, x_t])
候选状态：h̃_t = tanh(W_h · [r_t ⊙ h_{t-1}, x_t])
隐藏状态：h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t

  3. 双向RNN

    • 结合前向和后向隐藏状态

    • 适用于序列两端信息都重要的任务

2.1.3 注意力机制基础#

注意力机制是Transformer的核心，它允许模型关注输入的不同部分。

基本概念：

Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V

Q：查询矩阵
K：键矩阵
V：值矩阵
d_k：键的维度

注意力分数计算：

  • Q与K的点积衡量相关性

  • softmax归一化为概率分布

  • 用V加权聚合信息

优势：

  • 稀疏关注：只关注重要部分

  • 并行计算：避免RNN的顺序依赖

  • 长距离依赖：直接连接相关位置

2.2 稠密表示学习#

2.2.1 Word2Vec#

Word2Vec是词嵌入的里程碑工作，将词映射为低维向量空间。

CBOW模型：

输入：上下文词 [w_{t-k}, ..., w_{t-1}, w_{t+1}, ..., w_{t+k}]
输出：中心词 w_t
目标：最大化上下文词预测中心词的概率

Skip-gram模型：

输入：中心词 w_t
输出：上下文词 [w_{t-k}, ..., w_{t-1}, w_{t+1}, ..., w_{t+k}]
目标：最大化中心词预测上下文词的概率

学习到的语义：

  • 词向量空间中，相似词语距离近

  • 可进行类比推理：king - man + woman ≈ queen

  • 词向量蕴含丰富的语义和句法信息

2.2.2 GloVe#

GloVe（全局向量）结合局部上下文和全局统计信息。

核心思想：

  • 共现矩阵捕获词的协同出现

  • 矩阵分解学习词向量

GloVe目标：

最小化以下损失：
f(x_ij) · (w_iᵀ·w_j + b_i + b_j - log x_ij)²

x_ij：词i和词j的共现次数
f(x_ij)：加权函数，减少高频词的影响
w_i, w_j：词向量
b_i, b_j：偏置

优势：

  • 结合局部上下文和全局统计

  • 训练速度更快

  • 在很多任务上表现优秀

2.2.3 BERT预训练模型#

BERT（Bidirectional Encoder Representations from Transformers）是预训练模型的里程碑。

BERT架构：

  • 基于Transformer Encoder

  • 双向上下文理解

  • 大规模预训练 + 任务微调

预训练任务：

  1. 掩码语言模型 (MLM)

    随机掩码15%的词
预测被掩码的词

  2. 下一句预测 (NSP)

    判断两句是否连续
增强句间关系理解

特点：

  • 深度双向上下文

  • 强大的语义理解

  • 通用性强，易于微调

2.3 序列建模#

2.3.1 Seq2Seq模型#

Seq2Seq（Encoder-Decoder）模型用于序列到序列的转换任务。

模型结构：

编码器：将输入序列编码为固定长度向量
解码器：从固定长度向量解码为目标序列

问题：

  • 固定长度向量无法承载长序列信息

  • 信息瓶颈

改进：

  • 使用注意力机制缓解信息瓶颈

  • Transformer完全基于注意力机制

2.3.2 Transformer之前的发展#

在Transformer之前，序列建模主要依赖RNN及其变体。

RNN的局限：

  • 难以并行计算

  • 长期依赖问题

改进方向：

  • 使用CNN处理序列

  • 引入注意力机制

  • 最终诞生Transformer

2.4 训练技巧#

2.4.1 正则化方法#

正则化防止过拟合，提高模型泛化能力。

1. L1/L2正则化

L_reg = λ · ||w||₁  或  L_reg = λ · ||w||₂²
λ：正则化系数

2. Dropout

随机丢弃部分神经元
训练时：p的神经元置0
测试时：使用所有神经元，权重缩放(1-p)

3. 早停 (Early Stopping)

在验证集上监控损失
当验证损失不再下降时停止训练

2.4.2 批归一化#

批归一化加速训练，提高模型稳定性。

原理：

μ_B = (1/m) Σ x_i
σ_B² = (1/m) Σ (x_i - μ_B)²
x̂_i = (x_i - μ_B) / √(σ_B² + ε)
y_i = γ · x̂_i + β

μ_B：batch的均值
σ_B²：batch的方差
γ, β：可学习的缩放和平移参数
ε：防止除零的小常数

优势：

  • 加速收敛

  • 允许更高的学习率

  • 减少对初始化的依赖

  • 稀疏激活

2.4.3 学习率调度#

学习率调度优化训练过程。

常见策略：

  1. 固定学习率：简单但可能欠优

  2. 学习率衰减：随训练衰减

    • Step decay

    • Exponential decay

    • Cosine annealing

  3. 预热 (Warm-up)

    前几个epoch从小学习率逐渐增加
防止初期不稳定

  4. 余弦退火

    学习率呈余弦曲线变化

2.4.4 梯度裁剪#

梯度裁剪防止梯度爆炸。

L2裁剪：

如果 ||g|| > θ，则缩放梯度：
g ← θ · g / ||g||

裁剪位置：

  • 优化器中

  • 损失函数后

  • RNN的梯度

本章小结#

本章介绍了现代深度学习的核心技术：

  1. CNN：卷积神经网络，擅长局部特征提取

  2. RNN：循环神经网络，处理序列数据

  3. 注意力机制：关注输入不同部分，是Transformer的基础

  4. Word2Vec：词嵌入，将词映射为向量

  5. GloVe：结合全局统计的词向量方法

  6. BERT：双向Transformer预训练模型

这些技术为理解和实现Transformer、大模型奠定了基础。

代码示例#

# 注意力机制实现
import torch
import torch.nn as nn
import torch.nn.functional as F

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def scaled_dot_product_attention(self, Q, K, V, mask=None):
        scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(self.d_k)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)

        attention = F.softmax(scores, dim=-1)
        output = torch.matmul(attention, V)
        return output, attention

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # 线性变换
        Q = self.W_q(query)  # (batch, seq_len, d_model)
        K = self.W_k(key)
        V = self.W_v(value)

        # 分割多头
        Q = Q.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        # 注意力计算
        x, attention_weights = self.scaled_dot_product_attention(Q, K, V, mask)
        x = x.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # 输出层
        return self.W_o(x), attention_weights

# LSTM实现
class LSTMCell(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.input_size = input_size
        self.hidden_size = hidden_size

        self.i_gate = nn.Linear(input_size + hidden_size, hidden_size)
        self.f_gate = nn.Linear(input_size + hidden_size, hidden_size)
        self.o_gate = nn.Linear(input_size + hidden_size, hidden_size)
        self.c_tilde = nn.Linear(input_size + hidden_size, hidden_size)

    def forward(self, x, h_prev, c_prev):
        # 合并输入和隐藏状态
        combined = torch.cat((x, h_prev), dim=-1)

        # 计算门
        i = torch.sigmoid(self.i_gate(combined))
        f = torch.sigmoid(self.f_gate(combined))
        o = torch.sigmoid(self.o_gate(combined))
        c_tilde = torch.tanh(self.c_tilde(combined))

        # 更新细胞状态
        c_next = f * c_prev + i * c_tilde
        h_next = o * torch.tanh(c_next)

        return h_next, c_next

# 批归一化示例
class BatchNorm1d(nn.Module):
    def __init__(self, num_features, eps=1e-5, momentum=0.1):
        super().__init__()
        self.eps = eps
        self.momentum = momentum

        self.gamma = nn.Parameter(torch.ones(num_features))
        self.beta = nn.Parameter(torch.zeros(num_features))

        self.register_buffer('running_mean', torch.zeros(num_features))
        self.register_buffer('running_var', torch.ones(num_features))

    def forward(self, x):
        if self.training:
            mean = x.mean(dim=0)
            var = x.var(dim=0, unbiased=False)

            # 更新running统计量
            self.running_mean = (1 - self.momentum) * self.running_mean + self.momentum * mean
            self.running_var = (1 - self.momentum) * self.running_var + self.momentum * var
        else:
            mean = self.running_mean
            var = self.running_var

        x_centered = x - mean
        x_normalized = x_centered / torch.sqrt(var + self.eps)

        return self.gamma * x_normalized + self.beta

运行示例：

# 测试注意力机制
import torch

batch_size = 4
seq_len = 10
d_model = 512
num_heads = 8

query = torch.randn(batch_size, seq_len, d_model)
key = torch.randn(batch_size, seq_len, d_model)
value = torch.randn(batch_size, seq_len, d_model)

attention = MultiHeadAttention(d_model, num_heads)
output, attn_weights = attention(query, key, value)

print(f"Query shape: {query.shape}")      # (4, 10, 512)
print(f"Output shape: {output.shape}")    # (4, 10, 512)
print(f"Attention weights shape: {attn_weights.shape}")  # (4, 8, 10, 10)

# 测试LSTM
lstm = LSTMCell(input_size=128, hidden_size=256)

x = torch.randn(5, 3, 128)  # (seq_len, batch_size, input_size)
h_prev = torch.randn(3, 256)
c_prev = torch.randn(3, 256)

for t in range(x.size(0)):
    h_next, c_next = lstm(x[t], h_prev, c_prev)
    h_prev, c_prev = h_next, c_next

print(f"Output shape: {h_prev.shape}")  # (3, 256)
