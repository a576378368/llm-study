## 2.1 推理优化

### 2.1.1 量化技术

量化通过降低模型权重和计算的精度来提升推理效率。

**量化基础概念**：

| 精度格式 | 位宽 | 内存占用 | 适用场景 |
|----------|------|----------|----------|
| FP32 | 32bit | 100% | 训练/高精度推理 |
| FP16 | 16bit | 50% | 加速推理 |
| BF16 | 16bit | 50% | 混合精度训练 |
| INT8 | 8bit | 25% | 量化推理 |
| INT4 | 4bit | 12.5% | 高压缩推理 |

```python

def static_quantize(model, calibration_data):
    """静态量化 - 需要校准数据"""
    # 1. 设置qconfig
    model.qconfig = torch.quantization.get_default_qconfig('fbgemm')
    torch.quantization.prepare(model, inplace=True)

    # 2. 校准
    model.eval()
    with torch.no_grad():
        for batch in calibration_data:
            model(batch)

    # 3. 转换
    quantized_model = torch.quantization.convert(model, inplace=False)
    return quantized_model
```

### 2.1.2 知识蒸馏

知识蒸馏将大模型的能力迁移到小模型。

**基础蒸馏**：

```python
class DistillationLoss(nn.Module):
    """蒸馏损失"""
    def __init__(self, temperature=4.0, alpha=0.5):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha

    def forward(self, student_logits, teacher_logits, labels):
        # 软损失
        soft_teacher = F.softmax(teacher_logits / self.temperature, dim=-1)
        soft_student = F.log_softmax(student_logits / self.temperature, dim=-1)
        loss_soft = -torch.sum(soft_teacher * soft_student, dim=-1).mean()

        # 硬损失
        loss_hard = F.cross_entropy(student_logits, labels)

        # 加权
        return self.alpha * (self.temperature ** 2) * loss_soft + (1 - self.alpha) * loss_hard
```

**特征蒸馏**：

```python
class FeatureDistillation:
    """特征图蒸馏"""
    def __init__(self, teacher, student, layer_mapping):
        self.teacher = teacher
        self.student = student
        self.layer_mapping = layer_mapping  # {student_layer: teacher_layer}

    def compute_feature_loss(self, student_hidden, teacher_hidden):
        """中间层特征损失"""
        loss = 0

        for s_layer, t_layer in self.layer_mapping.items():
            # L2损失或cosine损失
            s_feat = student_hidden[s_layer]
            t_feat = teacher_hidden[t_layer]

            # 维度对齐（如有必要）
            if s_feat.shape != t_feat.shape:
                s_feat = self.project(s_feat, t_feat.shape[-1])

            loss += F.mse_loss(s_feat, t_feat)

        return loss

    def project(self, x, target_dim):
        """投影层对齐维度"""
        return torch.nn.functional.linear(x, self.projection_matrix)
```

### 2.1.3 模型剪枝

剪枝移除不重要的权重以减少模型大小。

**幅度剪枝**：

```python
class MagnitudePruner:
    """幅度剪枝"""
    def __init__(self, sparsity=0.5):
        self.sparsity = sparsity

    def prune(self, model):
        """对线性层进行幅度剪枝"""
        for name, module in model.named_modules():
            if isinstance(module, torch.nn.Linear):
                weight = module.weight.data
                threshold = torch.quantile(torch.abs(weight), self.sparsity)
                mask = torch.abs(weight) > threshold
                module.weight.data *= mask.float()
                print(f"Layer {name}: {mask.sum()}/{mask.numel()} kept")

        return model
```

**结构化剪枝**：

```python
class StructuredPruner:
    """结构化剪枝 - 按神经元/注意力头剪枝"""
    def __init__(self, model):
        self.model = model

    def prune_attention_heads(self, num_heads_to_prune):
        """剪枝注意力头"""
        for layer in self.model.model.layers:
            # 计算每个头的重要性（基于注意力熵）
            importance_scores = self.compute_head_importance(layer)

            # 剪枝最不重要的头
            heads_to_remove = torch.argsort(importance_scores)[:num_heads_to_prune]

            # 创建新模块
            new_attn = self.create_pruned_attention(
                layer.self_attn,
                heads_to_remove
            )
            layer.self_attn = new_attn

    def compute_head_importance(self, layer):
        """计算注意力头重要性"""
        importance = []
        num_heads = layer.self_attn.num_heads

        for head in range(num_heads):
            # 使用梯度或注意力分布作为重要性指标
            # 简化示例
            score = torch.norm(layer.self_attn.q_proj.weight[head]).item()
            importance.append(score)

        return torch.tensor(importance)
```

### 2.1.4 Flash Attention

Flash Attention是一种高效的注意力机制实现，通过IO感知和tiling技术显著降低计算复杂度。

**核心思想**：

标准Attention计算复杂度为 $O(N^2)$，Flash Attention通过分块计算将复杂度保持不变，但显著降低显存占用。

**Forward Pass**：

```python
import torch

def flash_attention_forward(Q, K, V, block_size=128):
    """
    Flash Attention前向传播（简化版）
    Q, K, V: (batch, heads, seq_len, head_dim)
    """
    batch, heads, seq_len, head_dim = Q.shape

    # 缩放因子
    scale = 1.0 / (head_dim ** 0.5)

    # 初始化输出
    output = torch.zeros_like(Q)

    # 分块计算
    for i in range(0, seq_len, block_size):
        # 获取当前块
        Q_block = Q[:, :, i:i+block_size, :]  # (batch, heads, block, head_dim)

        # 计算当前块的注意力输出
        block_output = block_attention(Q_block, K, V, scale, block_size)
        output[:, :, i:i+block_size, :] = block_output

    return output

def block_attention(Q_block, K, V, scale, block_size):
    """块内注意力计算"""
    seq_len = K.shape[2]

    # 累加器和归一化因子
    m = torch.full((Q_block.shape[0], Q_block.shape[1], Q_block.shape[2], 1),
                   float('-inf'), device=Q_block.device)
    l = torch.zeros(Q_block.shape[0], Q_block.shape[1], Q_block.shape[2], 1, device=Q_block.device)
    output = torch.zeros_like(Q_block)

    for j in range(0, seq_len, block_size):
        K_block = K[:, :, j:j+block_size, :]
        V_block = V[:, :, j:j+block_size, :]

        # 计算s和l
        s = torch.einsum('bhnd,bhmd->bhnm', Q_block, K_block) * scale

        # Safe exponential
        m_next = torch.maximum(m, torch.max(s, dim=-1, keepdim=True)[0])
        l_next = torch.exp(m - m_next) * l + torch.exp(s - m_next).sum(dim=-1, keepdim=True)

        # 更新输出
        p = torch.exp(s - m_next)
        output = (output * torch.exp(m - m_next) * l + torch.einsum('bhnm,bhmd->bhnd', p, V_block)) / l_next

        m = m_next
        l = l_next

    return output
```

```cuda
// 矩阵乘法CUDA核函数
__global__ void matrixMultiply(float *A, float *B, float *C,
                               int M, int N, int K) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;

    if (row < M && col < N) {
        float sum = 0.0f;
        for (int k = 0; k < K; k++) {
            sum += A[row * K + k] * B[k * N + col];
        }
        C[row * N + col] = sum;
    }
}

// 调用
dim3 block(16, 16);
dim3 grid((N + 15) / 16, (M + 15) / 16);
matrixMultiply<<<grid, block>>>(A, B, C, M, N, K);
```

**共享内存优化**：

```cuda
__global__ void matrixMultiplyShared(float *A, float *B, float *C,
                                      int M, int N, int K) {
    __shared__ float sA[16][16];
    __shared__ float sB[16][16];

    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;

    float sum = 0.0f;

    for (int i = 0; i < (K + 15) / 16; i++) {
        // 加载到共享内存
        if (row < M && (i * 16 + threadIdx.x) < K) {
            sA[threadIdx.y][threadIdx.x] = A[row * K + i * 16 + threadIdx.x];
        } else {
            sA[threadIdx.y][threadIdx.x] = 0.0f;
        }

        if (col < N && (i * 16 + threadIdx.y) < K) {
            sB[threadIdx.y][threadIdx.x] = B[(i * 16 + threadIdx.y) * N + col];
        } else {
            sB[threadIdx.y][threadIdx.x] = 0.0f;
        }

        __syncthreads();

        // 计算
        for (int j = 0; j < 16; j++) {
            sum += sA[threadIdx.y][j] * sB[j][threadIdx.x];
        }

        __syncthreads();
    }

    if (row < M && col < N) {
        C[row * N + col] = sum;
    }
}
```

**内存池管理**：

```python
class MemoryPool:
    """GPU内存池"""
    def __init__(self, total_size):
        self.total_size = total_size
        self.available = total_size
        self.blocks = []

    def allocate(self, size):
        """分配内存块"""
        # 搜索合适的空闲块
        for i, block in enumerate(self.blocks):
            if block.size >= size and not block.allocated:
                block.allocated = True
                self.available -= size
                return block

        # 创建新块
        block = self._create_block(size)
        self.blocks.append(block)
        return block

    def free(self, block):
        """释放内存块"""
        block.allocated = False
        self.available += block.size
```

### 2.3.2 显存管理

**梯度累积优化**：

```python
class GradientAccumulation:
    """梯度累积以节省显存"""
    def __init__(self, model, accumulation_steps=4):
        self.model = model
        self.accumulation_steps = accumulation_steps
        self.accumulated_grads = {}

    def step(self, batch_idx, batch):
        """累积梯度"""
        outputs = self.model(**batch)
        loss = outputs.loss / self.accumulation_steps
        loss.backward()

        if (batch_idx + 1) % self.accumulation_steps == 0:
            # 执行优化步骤
            self.optimizer.step()
            self.optimizer.zero_grad()

            # 释放累积的梯度
            self.accumulated_grads = {}
```

**混合精度训练**：

```python
class MixedPrecisionManager:
    """混合精度管理器"""
    def __init__(self, enabled=True):
        self.enabled = enabled
        self.scaler = torch.cuda.amp.GradScaler() if enabled else None

    def forward(self, model, inputs):
        """混合精度前向"""
        if self.enabled:
            with torch.cuda.amp.autocast():
                return model(inputs)
        return model(inputs)

    def backward(self, loss):
        """混合精度反向"""
        if self.enabled:
            self.scaler.scale(loss).backward()
        else:
            loss.backward()

    def step(self, optimizer):
        """混合精度优化"""
        if self.enabled:
            self.scaler.unscale_(optimizer)
            torch.nn.utils.clip_grad_norm_(optimizer.parameters(), 1.0)
            self.scaler.step(optimizer)
            self.scaler.update()
        else:
            optimizer.step()
```

**连续批处理（Continuous Batching）**：

```python
class ContinuousBatcher:
    """连续批处理调度器"""
    def __init__(self, model, max_batch_size=32):
        self.model = model
        self.max_batch_size = max_batch_size

        self.pending_requests = []
        self.running_batches = []

    def add_request(self, request):
        """添加请求"""
        self.pending_requests.append(request)

    def schedule(self):
        """调度执行批"""
        # 合并等待中的请求
        while len(self.pending_requests) >= self.max_batch_size:
            batch = self.pending_requests[:self.max_batch_size]
            self.pending_requests = self.pending_requests[self.max_batch_size:]

            # 启动批处理
            self.running_batches.append(self.execute_batch(batch))

        # 检查完成
        self._check_completed_batches()

    async def execute_batch(self, batch):
        """执行一批请求"""
        # 提取prompts
        prompts = [r['prompt'] for r in batch]

        # 生成
        outputs = self.model.generate(prompts)

        # 完成请求
        for i, request in enumerate(batch):
            request['future'].set_result(outputs[i])

        return batch
```

```python
        """并行生成"""
        chunk_size = len(prompts) // self.num_workers
        chunks = [prompts[i:i+chunk_size] for i in range(0, len(prompts), chunk_size)]

        with ProcessPoolExecutor(max_workers=self.num_workers) as executor:
            results = list(executor.map(self.generate_chunk, chunks))

        return [r for chunk in results for r in chunk]

    def generate_chunk(self, prompts):
        """生成一个chunk"""
        outputs = []
        for prompt in prompts:
            out = self.model.generate(prompt)
            outputs.append(out)
        return outputs
```

```python
                self.stream = torch.cuda.Stream()

                first = True
                for next_batch in self.loader:
                    with torch.cuda.stream(self.stream):
                        next_batch = {k: v.cuda() for k, v in next_batch.items()}

                    if not first:
                        yield batch
                    batch = next_batch
                    first = False

                yield batch

        return PrefetchLoader(data_loader)
```

## 本章小结

本章介绍了模型优化的核心技术：

1. **推理优化**：
   - **量化**：GPTQ、AWQ等权重量化方法显著减少模型大小
   - **蒸馏**：知识蒸馏将大模型能力迁移到小模型
   - **剪枝**：移除不重要的权重或神经元
   - **Flash Attention**：IO优化的注意力实现

2. **加速技术**：
   - **CUDA编程**：硬件级并行计算
   - **TensorRT**：NVIDIA高性能推理引擎
   - **ONNX Runtime**：跨平台推理优化

3. **内存优化**：
   - **KV Cache**：分页缓存管理
   - **显存管理**：梯度累积和混合精度
   - **内存池**：高效的GPU内存分配

4. **性能调优**：
   - **连续批处理**：提高GPU利用率
   - **多进程**：并行推理
   - **算子融合**：减少计算开销
